krukow.copilot-sdk.client

CopilotClient - manages connection to the Copilot CLI server.

client

(client)(client opts)

Create a new CopilotClient.

Options: - :cli-path - Path to CLI executable (default: “copilot”) - :cli-args - Extra arguments for CLI - :cli-url - URL of existing server (e.g., “localhost:8080”) - :cwd - Working directory for CLI process - :port - TCP port (default: 0 for random) - :use-stdio? - Use stdio transport (default: true) - :log-level - :none :error :warning :info :debug :all - :auto-start? - Auto-start on first use (default: true) - :auto-restart? - Auto-restart on crash (default: true) - :env - Environment variables map

connect-with-streams!

(connect-with-streams! client in out)

Connect to a server using pre-existing input/output streams. For testing purposes only.

create-session

(create-session client)(create-session client config)

Create a new conversation session.

Config options: - :session-id - Custom session ID - :model - Model to use (e.g., “gpt-5”) - :tools - Vector of tool definitions - :system-message - System message config - :available-tools - List of allowed tool names - :excluded-tools - List of excluded tool names - :provider - Custom provider config (BYOK) - :on-permission-request - Permission handler function - :streaming? - Enable streaming - :mcp-servers - MCP server configs - :custom-agents - Custom agent configs

Returns a CopilotSession.

delete-session!

(delete-session! client session-id)

Delete a session and its data from disk.

force-stop!

(force-stop! client)

Force stop the CLI server without graceful cleanup.

get-last-session-id

(get-last-session-id client)

Get the ID of the most recently updated session.

list-sessions

(list-sessions client)

List all available sessions. Returns a vector of session metadata maps.

notifications

(notifications client)

Get the channel that receives non-session notifications. Notifications are dropped if the channel is full.

ping

(ping client)(ping client message)

Ping the server to check connectivity. Returns {:message :timestamp :protocol-version}.

resume-session

(resume-session client session-id)(resume-session client session-id config)

Resume an existing session by ID.

Config options (same as create-session except session-id/model): - :tools - :provider - :streaming? - :on-permission-request - :mcp-servers - :custom-agents

Returns a CopilotSession.

start!

(start! client)

Start the CLI server and establish connection. Blocks until connected or throws on error.

state

(state client)

Get the current connection state.

stop!

(stop! client)

Stop the CLI server and close all sessions. Returns a vector of any errors encountered during cleanup.
